Phase 1 人声模块详解

模块 职责 技术
人声分离 从用户上传的歌曲中分离出干净的人声（干声）
现有 Demucs mock → 接入真实模型		 UVR / Demucs
声纹验证 用户实时录音，与上传的干声比对声纹特征
确保"训练的是我自己的声音"		 SpeakerNet / ECAPA
RVC 训练 用分离的干声训练个人声音模型
渐进式：20 epoch → 100 epoch → 200 epoch		 RVC (HuBERT + VITS)
RVC 推理 输入参考人声（用户演唱/哼唱），输出目标声音的歌声
保留腔调，替换音色		 RVC inference
声音模型管理 用户可训练多个声音模型，切换使用
每个模型绑定一个用户，不可转让		 CRUD + 文件管理

关键设计决策

模型加载策略（与器乐管线一致）

RTX 5080 16GB 显存有限，采用顺序加载/卸载：Demucs → RVC训练 → 卸载 → MusicGen → 卸载。每次只有一个模型驻留显存。

渐进式训练（解决速度焦虑）

RVC 训练 20 epochs 即出初版（~2min），100 epochs 标准版（~8min），200 epochs 精品版（~20min）。每个 checkpoint 独立存储，用户随时可用低 epoch 版本。

声音模型归属

每个 VoiceModel 绑定到 user_id，通过声纹验证确保上传者是本人。禁止跨用户共享/导出模型文件。
