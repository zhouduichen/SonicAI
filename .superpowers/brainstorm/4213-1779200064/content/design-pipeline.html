设计 Section 3：RVC 训练管线

从上传歌曲到可用声音模型的完整流程

Phase 1 完整数据流

用户上传歌曲 (MP3/WAV/FLAC)
│
Step 1 — 人声分离 (Celery Task)
UVR/Demucs 分离人声轨和伴奏轨 → 保存干声 .wav
耗时: ~2min | 显存: ~4GB | 完成后卸载模型
│
Step 2 — 音频预处理
切片 (slicing) + 响度归一化 + 静音段裁剪
耗时: ~30s | CPU操作
│
Step 3 — 特征提取
HuBERT 提取音素级特征 → 保存 .pt 特征文件
耗时: ~2min | 显存: ~3GB | 完成后卸载HuBERT
│
Step 4 — RVC 渐进式训练
加载 VITS 基础模型 → Fine-tune on 用户干声
┌─────────────────────────────────────────────┐
│ 20 epochs → Preview → 保存 checkpoint │
│ 100 epochs → Standard → 保存 checkpoint │
│ 200 epochs → Premium → 保存 checkpoint │
└─────────────────────────────────────────────┘
耗时: 2min → 8min → 20min | 显存: ~8GB | 完成后卸载VITS
│
Step 5 — RVC 推理（用户触发）
用户上传参考人声（自己唱/哼的歌） → RVC 转换为目标声音
耗时: ~10-30s/分钟音频 | 显存: ~5GB
│
输出: 克隆声音演唱的歌声 .wav

关键实现细节

RVC 模型集成方式

RVC 是独立的 GitHub 项目，不是 pip 包。集成策略：

# 方式1：Git子模块引入核心推理代码
backend/app/services/rvc/ ← RVC项目的 infer/ 和 train/ 目录

# 方式2：subprocess调用RVC脚本
Python subprocess执行 RVC 的 infer.py / train.py

→ 推荐方式1（子模块）：更可控，可复用RVC的HuBERT预处理等模块
Celery 任务编排

使用 Celery Chain 串联，每个步骤独立可重试：

chain(
  separate_vocals.s(audio_id),
  preprocess_vocals.s(),
  extract_hubert_features.s(),
  train_rvc_progressive.s(epoch_targets=[20,100,200])
)
