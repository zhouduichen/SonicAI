设计 Section 2：数据模型 & API

Phase 1 新增的数据表和接口

新增数据模型

# VoiceModel — 声音模型

id: UUID
user_id: FK → User
name: str (用户命名的声音名称，如 "我的原声")
source_audio_id: FK → AudioAsset (训练的源音频)
checkpoint_path: str (RVC 模型权重文件路径)
config_path: str (RVC 模型配置文件)
status: enum[pending, preprocessing, training, ready, failed]
epoch: int (当前训练到第几个 epoch)
quality_tier: enum[preview, standard, premium]
duration_seconds: float (训练音频总时长)
created_at, updated_at

# VocalGeneration — 人声生成记录

id: UUID
user_id: FK → User
voice_model_id: FK → VoiceModel
reference_audio_id: FK → AudioAsset (参考人声，用户的演唱)
output_path: str (生成的人声文件路径)
status: enum[pending, processing, completed, failed]
duration_seconds: float
created_at

新增 API 端点

方法 路径 说明 认证
POST /api/v1/voice/train 提交声音训练任务（传入已上传的音频ID） JWT
GET /api/v1/voice/status/{model_id} 查询训练进度（epoch数、当前质量档位） JWT
GET /api/v1/voice/models 获取用户的所有声音模型列表 JWT
DELETE /api/v1/voice/models/{model_id} 删除声音模型及文件 JWT
POST /api/v1/voice/sing 用人声模型 + 参考音频生成歌声 JWT
GET /api/v1/voice/generations 获取人声生成历史列表 JWT

训练请求体示例

POST /api/v1/voice/train
{
  "audio_asset_id": "uuid-of-uploaded-song",
  "name": "我的声音",
  "quality_target": "premium" // preview | standard | premium
}

→ Celery Task: train_voice_model.delay(...)
→ 返回: { "model_id": "...", "status": "preprocessing" }

生成请求体示例

POST /api/v1/voice/sing
{
  "voice_model_id": "uuid-of-trained-model",
  "reference_audio_id": "uuid-of-sung-reference"
}

→ Celery Task: infer_rvc_vocals.delay(...)
→ 返回: { "generation_id": "...", "status": "processing" }
